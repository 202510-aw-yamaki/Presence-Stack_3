# Web面接ガイド 改訂版実装指示書 v3

作成目的: 既存サイトの体験資産を壊さずに、章トップと統合ページを分離した新サイトマップへ再編し、旧ファイルを残したまま実装を進めるための実装指示書。

---

## 0. 今回の改修の結論

今回の改修は、**章トップページと統合ページを分けること**、**既存のJS演出とモーダル挙動を復活させること**、**既存テキストを簡略化せずに保持すること**が中心です。

この改修は、単なる見た目調整ではなく、以下の方針で実装してください。

1. 旧ファイルは削除しない
2. 章トップページは章トップページとして残す
3. 統合ページは新規ファイルとして実装する
4. 統合ページ上部に章トップと同じ目次サムネイルを重複配置しない
5. 既存 `common-navigation.js` の遷移演出・ヘッダー生成・前後導線・スワイプを活かす
6. 旧個別ページにあった「テキストクリック / 画像クリックで開始面が変わるモーダル」を統合ページでも復活させる
7. `テキストの見本_抽出.txt` と旧HTMLの文言を**要約・言い換え・簡略化せず**に移植する

---

## 1. 絶対条件（非交渉）

### 1-1. テキスト簡略化禁止

- 既存文言を短く言い換えない
- 原文の文体を変えない
- 箇条書きを減らさない
- ノートを1段落に圧縮しない
- 統合ページ化のために、元ページの「カード」「ノート」「補足」「注意点」を消さない
- 旧ページ側にある補足文や注意文を、統合ページで1行要約に置き換えない

### 1-2. 旧ファイル残置

以下は**削除禁止**。

- 章トップ: `pre-interview-checks.html` / `materials-preparation.html` / `chatgpt-support.html` / `closing.html`
- 旧詳細: `usage-device.html` / `audio-camera-desktop.html` / `environment-notifications.html` / `platform-overview.html` / `zoom.html` / `google-meet.html` / `microsoft-teams.html`
- 第2章旧詳細: `company-information.html` / `resume-career-history.html` / `portfolio.html` / `curriculum-learning-log.html` / `company-questionnaire.html`
- 第3章旧詳細: `job-understanding.html` / `reflection-memo.html` / `mock-qa.html` / `mock-interview.html`

### 1-3. 既存体験資産維持

- 共通ヘッダー / フッター生成は `common-navigation.js` を基準に維持
- ページ遷移演出（SYSTEM TRANSFER）は維持
- スワイプ遷移は維持
- モーダル内のテキスト面 / 画像面の往復UIは維持
- 章トップの YouTube 風サムネイル導線は維持

## 3. 現在の問題点と修正方針

### 3-1. 統合ページ上部に章トップと同じ目次が残っている

**問題**
- 統合ページなのに上部に章トップと同型のサムネイル目次が残っている
- その下にローカル目次もあるため、導線が二重化している

**修正方針**
- `chapter-1-integrated.html` / `chapter-2-integrated.html` / `chapter-3-integrated.html` の上部から、章トップと同型のサムネイル目次ブロックを削除する
- 統合ページ上部は以下のみとする
  - タイトルカード
  - 導入カードまたはノート
  - 追従ヘッダー右上のハンバーガーメニュー
- 統合ページ内移動は、**ヘッダー右端のハンバーガーメニュー**から行う

### 3-2. 各ページに背景色がなく、雰囲気が平たい

**修正方針**
- 既存のカード色は保ちながら、ページ全体のベース背景を章ごとに柔らかい色へ調整する
- 強い彩度は避け、目に優しい色合いにする

推奨トーン:
- 第1章: 淡い青緑〜グレイッシュグリーン
- 第2章: 淡いアイボリー〜セージ
- 第3章: 淡いブルーグレー〜ラベンダーグレー
- 第4章: 淡いクリーム〜ウォームグレー

実装方針:
- `body` の背景を章別クラスで切り替える
- `slide-page` は白すぎない半透明面を維持
- カードの可読性を下げない

### 3-3. ヘッダーが追従でない

**修正方針**
- `slide-header` を `position: sticky; top: 0; z-index: 30;` にする
- スクロール時に軽いシャドウを付与
- 既存の `body.is-scrolled .slide-header` を活用し、視認性を高める
- 統合ページでは、ヘッダー右端にハンバーガーボタンを置く

### 3-4. `Sec.Top` チップが不要

**修正方針**
- 統合ページでは `Sec.Top` チップを非表示
- 章トップと index では既存導線を維持してよいが、統合ページでは右端ハンバーガーへ置換する
- ハンバーガーメニュー内に、同一ページ内アンカー一覧を出す

### 3-5. 1-4 は第1章の配下導線に寄せたい

**問題**
- `platform-detail-integrated.html` が章送りの連番対象になっている

**修正方針**
- `common-navigation.js` の `pages` 定義から、`chapter-1-integrated.html` → `platform-detail-integrated.html` → `materials-preparation.html` という直列章送りをやめる
- 第1章統合ページの 1-4 セクション内だけから、`platform-detail-integrated.html` へ入る導線を作る
- `platform-detail-integrated.html` 側の戻り導線は「第1章 1-4 へ戻る」を主とする

### 3-6. テキスト / 画像クリックで開始面が変わるモーダルが消えている

**修正方針**
- 統合ページでも、各セクション内の「テキスト + 画像」を1ユニットとして扱う
- テキスト側クリック時はモーダルのテキスト面から開始
- 画像側クリック時はモーダルの画像面から開始
- モーダル内は `前へ / 次へ` でテキスト面と画像面を往復できる構成へ戻す

### 3-7. 第4章「最後に」が殺風景

**修正方針**
- 「必須」「推奨」を単なる文章カードで終わらせない
- 右側または隣接カラムに、チェックシート風ビジュアルを配置する
- クリックでモーダルを開き、本文確認 + ダウンロード導線を出す
- 最下部に、振り返りメモテンプレートのダウンロードエリアを設ける

## 4. 実装方針（共通）

### 4-1. 章トップページの役割

章トップページは**目次ページ**であり、統合本文ページではない。

- 上部タイトル
- 章の導入文
- YouTube風サムネイル目次
- 各サムネイルから統合ページの該当アンカーへ遷移

### 4-2. 統合ページの役割

統合ページは**本文ページ**であり、目次ページではない。

- 上部に章トップと同じサムネイル目次は置かない
- 各セクションはカード単位で縦に並ぶ
- ヘッダー右端ハンバーガーメニューから各セクションへ移動
- 1セクション内は「左テキスト / 右画像2枚 or 特別レイアウト」
- モーダル復活

### 4-3. 旧個別ページの扱い

- 旧詳細ページは残す
- 統合ページ実装時の**文言参照元 / UI参照元**とする
- 新導線では主役にしないが、削除しない

## 5. `common-navigation.js` 改修指示

### 5-1. 方針

既存の `common-navigation.js` は、共通ヘッダー / フッター、前後リンク、スワイプ、ページ遷移演出を持っているため、**簡略化せず活かす**。

### 5-2. 必須修正

1. 統合ページでは `Sec.Topへ` を出さない
2. 統合ページでは右端にハンバーガーボタンを追加する
3. ハンバーガーメニューの中身は、そのページのセクションアンカー一覧にする
4. 第1章 1-4 詳細ページを、章送り `next` の直列から外す
5. `platform-detail-integrated.html` の戻り先は `chapter-1-integrated.html#section-platform-overview`
6. 章トップ / index / closing は既存に近い前後導線を維持

### 5-3. ハンバーガーメニュー仕様

- 右端にアイコンボタンを配置
- クリックでドロップダウン or オフキャンバスメニューを開く
- メニュー項目例（第2章統合）
  - 2-1 応募先企業の情報および求人情報
  - 2-2 自身の履歴書・職務経歴書
  - 2-3 自身のポートフォリオ
  - 2-4 職業訓練校のカリキュラムと学習記録
  - 2-5 企業アンケートも有効活用

### 5-4. ヘッダー追従

- `slide-header` は sticky
- 追従時に背景をやや不透明にする
- 遷移演出の `z-index` と競合しないよう注意

## 7-E. 第4章 `closing.html`

### 7-E-1. 見せ方の再設計

現状の文章量はあるが、見た目が単調。以下へ改修する。

**上段**
- 左: 「必須」チェックリストカード
- 右: チェックシート風画像カード
- クリックでモーダル

**中段**
- 左: 「推奨」チェックリストカード
- 右: チェックシート風画像カード
- クリックでモーダル

**下段**
- 全幅: 振り返りメモテンプレートエリア
- 画像プレビューまたはテキストプレビュー
- クリックでモーダル
- ダウンロードボタン配置

### 7-E-2. ダウンロード仕様

最低実装:
- 必須チェックシート: PNG / PDF / HTML印刷用のいずれか
- 推奨チェックシート: PNG / PDF / HTML印刷用のいずれか
- 振り返りメモテンプレート: CSV or TXT ダウンロード

発展実装（できれば）:
- 開始日 / 終了日を date input で指定
- 訓練期間の日付行を自動展開
- 出力項目は `日付 / 授業内容 / 一行メモ`
- CSV ダウンロード

### 7-E-3. 実現可否の判断

- **CSV / TXT のテンプレートダウンロードは実装可能**
- **ブラウザのみで完結する簡易ジェネレーターも実装可能**
- **Excelファイルそのものを綺麗に生成する実装は今回必須ではない**
- 工数が厳しい場合は、CSV / TXT + 印刷用HTMLで可

## 8. テキスト移植ルール

### 8-1. 正本の優先順位

1. `テキストの見本_抽出.txt`
2. 旧個別HTML
3. 旧統合前のノート / 補足 / 追加文

### 8-2. 移植時の原則

- 原文をそのまま使う
- 文章順も可能な限り維持
- 箇条書きを説明文へ畳み込まない
- 「カード」単位を維持する
- 「ノート」単位を維持する
- 元ページの注意喚起は削除しない
- 統合ページ化しても、「1セクション = 1枚の短縮版」にはしない

### 8-3. 旧ページから拾うべきUI構造

- `usage-device.html`
- `audio-camera-desktop.html`
- `environment-notifications.html`
- `platform-overview.html`
- `zoom.html`

- `company-information.html`
- `resume-career-history.html`

自分の振り返り内容を活かしながら、よりよい回答の仕方

タイトル: 3-4. ChatGPTと模擬面接を行う
カード: ChatGPTと模擬面接をして、自身を分析しましょう
カード: SWOT分析
SWOT分析とは自身の内部環境（強み・弱み）と外部環境（機会・脅威）を
4つの視点（S・W・O・T）で整理し、戦略策定に役立てるフレームワークです。
自身の【強み/機会】といったプラス要因をしっかりと把握し、
【弱み/脅威】のマイナス要因の払拭をしていきましょう。
そして面接においてプラス要因はとても大事ですが、
マイナスの要因を自身の伸びしろに変換してアピールできることが大事です。
カード: WCV（キャリアの3要素）
自己調整学習（Will-Can-Value）の視点も大事です。
・Will：やりたいこと・意思（何に挑戦したいか）
・Can：できること・能力（何ができるか）
・Value：大切にしたい価値観（譲れない条件）
この軸がないと面接官側からは「この人はどこでもいいのかな」という人間にみられてしまいます。
どんな人にも輝ける場所があり、そして譲ってはいけない価値観も大事に個を確立しましょう。
ノート: 学びを整理する考え方
職業訓練校での学びを整理する際は、単に「何を受講したか」を記録するだけでなく、
「どのような段階で理解が深まっているか」という視点を持つことも有効です。
その考え方の一つとして、守破離があります。
守破離とは、学びや成長の過程を、次の3段階で捉える考え方です。
・守：まずは型を守り、基本を習得する段階
・破：基本を土台にしながら、自分なりに工夫し、応用していく段階
・離：基本を踏まえたうえで、自分なりのやり方や強みを確立していく段階
