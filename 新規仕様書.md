# Web面接ガイド 改修指示書

## 0. この指示書の目的
本指示書は、既存の `Web面接ガイド` を、**現在の世界観・文言・導線の良さを残したまま**、以下の状態へ再編するための実装指示をまとめたものです。

- レベル0 `index.html`
- レベル1 各章トップ兼・統合ロングページ
- レベル2 各章内セクションを**1ページ内アンカー移動**へ統合
- レベル3 プラットフォーム詳細（Zoom / Google Meet / Microsoft Teams）も**1ページへ統合**
- 右カラム画像を**縦積み2枚・各1:1固定**へ統一
- テキストは**既存文言を減らさず・文体を変えず**、不足がある箇所だけ追記する
- 実装は**レスポンシブ**かつ**セマンティックな Bootstrap ベース**へ寄せる

---

## 1. まず押さえる前提

### 1-1. 現在の実装前提
現行サイトは、章トップ・詳細ページ・共通ヘッダー / フッター・前後ページ・ページ遷移演出を持つ静的サイトとして構成されている。`README.md` では、全ページ共通のヘッダー / フッター、前へ / 次へ / 章トップ / Index のナビゲーション、ページ遷移演出、章構成に合わせたフォルダ分割が既存仕様として整理されている。`js/common-navigation.js` が共通ヘッダー / フッターの差し込みとページごとの導線定義を担っているため、今回の改修は HTML だけではなく、**導線定義とページ粒度の見直しを含む**。fileciteturn3file15 fileciteturn3file16

### 1-2. 現在のレイアウト前提
章トップは、`common.css` 上で `gallery-grid` と `thumb-media` により、**YouTube風サムネイル一覧**として組まれており、この見せ方は維持対象である。`thumb-media` は `aspect-ratio: 16 / 9` を前提に構成されているため、章トップのサムネイル表現は崩さない。fileciteturn3file7

詳細ページは `section_common.css` により、`check-body` を **左テキスト / 右ビジュアル** の 2 カラムで構成している。現状は `grid-template-columns: minmax(0, 7fr) minmax(0, 5fr)`、右側画像は `object-fit: cover` を基準にしているため、ページごとに画像比率がぶれやすい。fileciteturn3file6 fileciteturn4file10

### 1-3. 比率崩れの具体例
完成済み参照ページの `audio-camera-desktop.html` では、同一ページ内でも `is-portrait` は 1:1、`is-landscape` は 16:9 としており、ページ単位・カード単位でアスペクト比の扱いが分かれている。これが「右側の画像の比率が不安定」に見える直接原因なので、今回の統合ページでは**例外クラスを撤廃**し、右カラムは **A画像 / B画像 の2枚縦積み、どちらも 1:1固定** にする。fileciteturn4file2

### 1-4. テキストソースの優先順位
テキストは以下の優先順位で扱うこと。

1. `テキストの見本_抽出.txt` を**文言の原本**として最優先
2. 既存 HTML の完成済み粒度 (`pre-interview-checks.html`, `audio-camera-desktop.html`) を**UI粒度の見本**として参照
3. 既存個別 HTML にのみある補助文は、原文維持で適宜統合
4. 不足がある場合のみ、**追記**を入れる

`テキストの見本_抽出.txt` には、index から closing までの本文が連続して整理されており、今回の「抜け落ちなく入れる」要求に対する主要根拠になる。fileciteturn2file1 fileciteturn2file2

## 3. 導線の再設計方針

## 3-1. index.html
`index.html` は現状どおり 4章への入口ページとし、YouTube風サムネイルを維持する。第1章のリンク先だけは、既存の `pre-interview-checks.html` ではなく、**新しい第1章統合ページ**へ差し替える。現在 index は Chapter 1〜4 のサムネイルリンクで構成されているため、その見え方は維持し、リンク先だけ更新する。fileciteturn1file2 fileciteturn4file8

## 3-2. レベル1ページの役割
各レベル1ページは、
- ページ上部：章タイトル、注釈、YouTube風目次サムネイル
- ページ下部：各セクションを縦に連結したロングページ

という **「目次＋本文一体型」** にする。

つまり、現行の `materials-preparation.html` や `chatgpt-support.html` のサムネイル部分は残し、その下に 2-1〜2-5 / 3-1〜3-4 をそのまま連結する構成に変える。現状の章トップは「各サムネイルをクリックすると各項目へ移動します」となっているので、この導線は**別ページ移動ではなく、同一ページ内アンカー移動**へ差し替える。fileciteturn3file13 fileciteturn3file17

## 3-3. サムネイルのリンク仕様
レベル1ページ上部のサムネイルリンクは、以下のように変更する。

- 変更前：`materials-preparation/company-information.html`
- 変更後：`#section-company-information`

ただし、**共通ページ遷移演出は同一ページアンカーには適用しない**。`common-navigation.js` では `href` が `#` 始まりのリンク、および同一ページ内の `url.hash` はトランジション対象外になっているため、この仕様はむしろ今回と相性が良い。アンカーリンクにすることで、既存のトランジション制御を壊さずに自動スクロール導線へ置き換えられる。fileciteturn3file5 fileciteturn3file8

## 3-4. 共通ナビの修正
`js/common-navigation.js` は現在、各サブページを `pages` オブジェクトで個別管理している。今回の統合後は、
- 削除される詳細ページエントリを整理
- 第1〜第4章のロングページだけを主導線として再定義
- `prev / next / total / current` のページ番号を再計算

すること。

特に現状は、第2章が `materials-preparation.html` → `company-information.html` → `resume-career-history.html` ... という直列遷移になっているため、このままでは統合後の前後ページが破綻する。統合後は「第2章の前へ / 次へ」は**章単位**で扱い、章内移動はアンカーリンク・目次サムネイル・スティッキーナビで補う。fileciteturn3file1 fileciteturn3file2 fileciteturn4file6

## 4. レイアウト実装方針（Bootstrap + セマンティック）

## 4-1. 採用する基本構造
レベル1・レベル2統合ページでは、Bootstrap 5 系の以下構造を基本とすること。

- `main.container-xxl`
- `section` 単位で章内セクションを区切る
- `row` / `col-lg-*` を用いた 2 カラム
- タイトルは `h1 > h2 > h3` の順で階層を厳守
- 目次は `nav` + `a`
- 注釈は `aside` ないし `div[role="note"]` 相当で意味づけ

推奨骨格：

```html
<main class="container-xxl py-4 py-lg-5">
  <header class="chapter-hero mb-4 mb-lg-5">
    ...章タイトル...
    ...注釈...
    ...YouTube風サムネイル目次...
  </header>

  <nav class="chapter-local-nav sticky-top" aria-label="第2章 セクション目次">
    ...アンカーリンク...
  </nav>

  <section id="section-company-information" class="chapter-section py-4 py-lg-5" aria-labelledby="heading-company-information">
    ...
  </section>
</main>
```

## 4-2. 章内セクションの2カラム
各セクション本文は、**左テキスト / 右画像2枚縦積み**で統一する。

### PC / タブレット（lg以上）
- 左：`col-lg-7` 〜 `col-xl-7`
- 右：`col-lg-5` 〜 `col-xl-5`
- 右カラムの中で `ratio ratio-1x1` を2つ並べる
- A画像 = 上部テキストと紐づく
- B画像 = 下部テキストと紐づく

### SP（md以下）
- 1カラム化
- テキストの直下に A画像、続いて次のテキスト、最後に B画像でも良い
- ただし、**視線順が壊れないこと**を最優先

## 4-3. 画像比率のルール
右側に配置する画像はすべて以下で統一する。

- 1セクションあたり 2枚
- 縦積み
- 各画像は 1:1
- `object-fit: cover` を基本
- UIスクリーンショットだけは、必要なら `object-fit: contain` を局所適用
- 画像が未確定の場所は **A / B の仮ラベル入りプレースホルダー** を置く

## 4-4. 今回追加するべきクラス例

```css
.section-visual-stack {
  display: grid;
  gap: 1rem;
}

.section-visual-card {
  border: 1px solid var(--line);
  border-radius: 18px;
  overflow: hidden;
  background: #fff;
}

.section-visual-card .ratio > img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

## 4-5. 既存 CSS の扱い
- `common.css` の章トップ・サムネイル表現は活かす
- `section_common.css` は全面破棄しない
- ただし、`check-card` ベースの「1カード1画像」発想は、今回の**統合ロングページ**には合わないため、**統合ページ専用のレイアウトクラスを追加**する
- `audio-camera-desktop.html` にある `is-portrait / is-landscape` のような個別例外は増やさない

## 6. 実装上の重要禁止事項

1. **既存文言を減らさないこと**
2. **既存文体を変えないこと**
3. 既存 HTML の本文を要約して短くしないこと
4. 章トップの YouTube 風サムネイルを崩さないこと
5. 右画像の比率をカードごとに変えないこと
6. レベル2統合後も、内容順は現行順序を守ること
7. Bootstrap 化しても、既存の柔らかい配色・余白感は残すこと
8. ページ遷移演出は**別ページ移動時のみ**とし、アンカー移動に誤適用しないこと

## 8. テキスト不足への対応ルール

## 8-1. 不足を感じた場合の足し方
ユーザー要望どおり、**文言を減らすことや文体を変えることは禁忌**である。したがって、文章量が不足して見える箇所は、以下のルールで追記する。

1. 既存本文をそのまま残す
2. 追記は本文の後ろに行う
3. 追記は「何をすると良いか」「なぜそれが必要か」を補う
4. 追記の語調は既存のやわらかい説明口調に合わせる
5. 箇条書きの項目を勝手に削減しない

## 8-2. 特に追記が有効な箇所
- 第1章
  - 背景・照明・目線の確認理由
  - 通知停止の情報漏えいリスク
  - デュアルモニター時の視線管理
- 第2章
  - なぜその資料が面接で効くのか
  - URL / PDF / README を分けて持つ意味
- 第3章
  - ChatGPT を使う際に「鵜呑みにしない」補足
  - 情報源を更新する必要性
- 第4章
  - 「準備の質が面接の進行を安定させる」ことの再確認

## 12. 最終指示
今回の改修で最も重要なのは、**「ページ数を減らすこと」ではなく、「準備の流れが自然に追えること」**である。既存サイトには、章トップ・共通ナビ・遷移演出・サムネイル導線といった良い要素がすでにある。したがって、完全に作り直すのではなく、

- 章トップの見せ方はそのまま
- 詳細情報はページ内統合
- 文言は原文維持
- 画像は 1:1 で統一
- Bootstrap でレスポンシブと保守性を上げる

という方向でまとめること。

特に、`テキストの見本_抽出.txt` にある本文は、今回の改修の核である。**既存文言を抜かず、減らさず、崩さず、必要分だけを足す**こと。これを最優先とする。fileciteturn2file1
