# Web面接ガイド 実装指示書 v2

## 0. この指示書の目的

本改修の目的は、**既存サイトの体験を壊さずに、情報設計のみを再編すること**です。

今回の実装で最も重要なのは、以下の4点です。

1. **レベル1（章トップ）とレベル2（統合本文ページ）を分離すること**
2. **レベル2は新規ファイルを追加して実装すること**
3. **既存の共通JS演出・ページ送り・トランジションを維持すること**
4. **`テキストの見本_抽出.txt` にある文言を抜け落ちなく保持すること**

このため、今回の改修は「章トップ兼ロングページ」にする実装ではありません。
**章トップは章トップのまま残し、その次の階層として新規の統合ロングページを追加する実装**にしてください。

---

## 1. 実装の前提条件

- 旧ファイルは削除しないこと
- 旧ファイルは参照用・バックアップ用として残してよい
- ただし、**メイン導線は新サイトマップに切り替える**こと
- `common-navigation.js` は既存演出を持つ版をベースにし、**簡略化版へ置き換えない**こと
- `common.css` / `section_common.css` は既存資産を活かしつつ拡張すること
- Bootstrap 5 系を使用すること
- ただし、Bootstrap で全面的に置き換えるのではなく、**既存デザイン・既存演出と共存させる**こと
- 文章は減らさないこと
- 文体を変えないこと
- 内容の補足は許可するが、補足は**原文の後ろに追記する**こと
- 補足によって原文の意味や温度感を変えないこと

## 2. 今回の完成形の理解

### 2-1. レベル構造

- **レベル0**：トップページ（`index.html`）
- **レベル1**：各章トップページ
- **レベル2**：各章の統合ロングページ（新規作成）
- **レベル3**：プラットフォーム詳細の統合ページ（新規作成）

### 2-2. 重要な設計原則

- `index.html` は各章トップへ遷移する
- 各章トップは「章の目次ページ」であり、本文の統合ページではない
- 各章トップの YouTube 風サムネイルは維持する
- 各章トップのサムネイルは、**新規の統合ページの該当セクションへ遷移**する
- 統合ページ内では、ローカル目次やヘッダー導線から該当セクションへスクロールできるようにする
- 第1章の `1-4. プラットフォーム概要` セクションから、レベル3の詳細統合ページへリンクする

## 5. ヘッダー / フッター / JS 演出に関する絶対条件

### 5-1. やってはいけないこと

- `common-navigation.js` を簡略版に差し替える
- ページ遷移演出を削除する
- 前へ / 次へ をフッターだけにする
- `resolvePageKey()` の判定基準を勝手に別プロジェクト名へ変更する
- `sessionStorage` を使った遷移状態の引き継ぎを消す
- スワイプ操作やキーボード導線を落とす

### 5-2. 守るべきこと

- 既存の遷移演出を持つ `common-navigation.js` をベースにする
- `pages` 定義に新規ページを追加する
- 旧ページ定義は残してよい
- ヘッダーには少なくとも以下を維持する
  - Index
  - Sec.Top
  - 前へ
  - 次へ
  - ページ番号 or 進捗
- フッターには前後ナビ・ページ番号・進捗バーを維持する
- スワイプとトランジションの挙動を維持する

### 5-3. pages 定義の考え方

`pages` には少なくとも以下を追加すること。

- `pre-interview-checks.html`

## 6. Bootstrap の使い方

今回の要件は「Bootstrapを使う」だが、**サイト体験までBootstrap標準へ寄せることは目的ではない**。

### 6-1. 方針

- Bootstrap 5 系を読み込む
- セマンティックな HTML を維持する
- レイアウトは Bootstrap の grid / container / row / col を活用する
- 比率は Bootstrap の `.ratio` / `.ratio-1x1` を使用してよい
- sticky は `position-sticky` と既存CSSの併用でよい
- ボタン、チップ、余白、レスポンシブ崩れ防止に Bootstrap を使う
- 既存のトランジション、トーン、カード、サムネイル演出は既存CSS優先

### 6-2. 具体的に Bootstrap を使う箇所

- 背景色
- レベル1トップのサムネイルグリッド
- レベル2統合ページの本文 2 カラム
- ローカル目次チップ群
- 右画像 2 段積み
- モバイル時 1 カラム落ち
- セクション間余白

## 7. 画像・レイアウト仕様

### 7-0. レベル0 / レベル1 共通

- 左側にYouTube 風サムネイル2カラム
- 右側に画像

### 7-1. レベル2 / レベル3 共通

- 左：本文
- 右：画像2枚縦積み
- 右画像は `A` と `B`
- どちらも **1:1**
- `A` は上段説明に紐づく
- `B` は下段説明に紐づく
- 画像は仮画像でもよいが、**A Image / B Image の仮文字だけで終わらせない**こと
- 仮実装でも最低限、説明キャプションを添えること

### 7-2. レスポンシブ

- PC：左本文 / 右画像スタック
- Tablet：左本文 / 右画像スタックでも可
- SP：本文の下に画像2枚
- SP時でも画像比率は 1:1 を維持する

### 7-3. チップ色指定（推奨）

以下の色分けルールを統一すること。

- Chapter / 共通説明：Blue
- 資料整理 / 手順系：Gold
- 学習・前向きな内容：Green
- AI活用 / 思考整理 / 分析：Purple
- 環境・補助・整理系：Slate

### 7-4. チップ文字

チップは説明補助であり、本文の代替ではない。本文を削ってチップで済ませないこと。

## 10. テキスト実装ルール

### 10-1. 最重要ルール

- `テキストの見本_抽出.txt` を**正本**として扱うこと
- 原文を削らない
- 原文を短くしない
- 原文の表現を整えすぎない
- 文体を変えない
- 箇条書きを別の要約文に変換しない
- 追記は可
- 追記する場合は、**原文のあとに補足として足す**

### 10-2. 統合ページでのテキスト配置ルール

- 章トップ文言は章トップに置く
- 個別ページ文言は統合ページの各セクションへ移す
- 1セクションに複数カードがあった場合、本文の順番は旧ページ順を維持する
- ノート文も必ず残す
- 注意書き（※）も削除しない

### 10-3. 追加が許可される箇所

- セクション冒頭のごく短いブリッジ文
- 画像キャプション
- ローカル目次の案内文
- CTA 文言
- レスポンシブで意味補助になる短文

### 13-20. 3-4 ChatGPTと模擬面接を行う

```text
タイトル: 3-4. ChatGPTと模擬面接を行う
カード: ChatGPTと模擬面接をして、自身を分析しましょう
カード: SWOT分析
SWOT分析とは自身の内部環境（強み・弱み）と外部環境（機会・脅威）を
4つの視点（S・W・O・T）で整理し、戦略策定に役立てるフレームワークです。
自身の【強み/機会】といったプラス要因をしっかりと把握し、
【弱み/脅威】のマイナス要因の払拭をしていきましょう。
そして面接においてプラス要因はとても大事ですが、
マイナスの要因を自身の伸びしろに変換してアピールできることが大事です。
カード: WCV（キャリアの3要素）
自己調整学習（Will-Can-Value）の視点も大事です。
・Will：やりたいこと・意思（何に挑戦したいか）
・Can：できること・能力（何ができるか）
・Value：大切にしたい価値観（譲れない条件）
この軸がないと面接官側からは「この人はどこでもいいのかな」という人間にみられてしまいます。
どんな人にも輝ける場所があり、そして譲ってはいけない価値観も大事に個を確立しましょう。
ノート: 学びを整理する考え方
職業訓練校での学びを整理する際は、単に「何を受講したか」を記録するだけでなく、
「どのような段階で理解が深まっているか」という視点を持つことも有効です。
その考え方の一つとして、守破離があります。
守破離とは、学びや成長の過程を、次の3段階で捉える考え方です。
・守：まずは型を守り、基本を習得する段階
・破：基本を土台にしながら、自分なりに工夫し、応用していく段階
・離：基本を踏まえたうえで、自分なりのやり方や強みを確立していく段階
```
